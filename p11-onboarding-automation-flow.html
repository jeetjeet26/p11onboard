Onboarding Automation Flow

This document maps the full automation architecture for P11creative's client onboarding system. It covers what fires automatically when a new client is created, what requires manual steps, and the recommended Zapier / Make.com (formerly Integromat) logic to connect the intake questionnaire to Accelo, Basecamp, Dropbox, Google Drive, and Softr.

⚡ Auto — Fully automated
✋ Manual — Human required
🔍 Verify — Auto + human check
🔮 Phase 2 — Future build
Automation Trigger
📋
Primary Trigger
Client Intake Questionnaire Submitted (Jotform / Softr Form)
All automations below fire from a single form submission event. The form submission is the single source of truth for the new client record.
↓
Zap 1 — Master Intake Handler (Zapier / Make.com)
⚡
Master New Client Zap
Fires once per form submission. Creates all records, folders, and project structures automatically.
1
TRIGGER — Jotform: New Submission
Watch the P11 Digital Marketing Intake Questionnaire form. On each new submission, pull all fields: community name, type, contacts, website URL, services selected, go-live date, platform access status.
TriggerJotform → Webhooks API
2
FORMATTER — Parse & Normalize Data
Clean up the community name (trim whitespace, title case). Format the go-live date as YYYY-MM-DD. Split services selected into a comma-delimited list. Extract primary contact name/email separately from billing contact.
Formatter step
3
FILTER — Is This a New Company in Accelo?
Search Accelo by company name. If a match is found → skip company creation, use existing Accelo Company ID. If no match → proceed to Step 4 to create new company. This prevents duplicate records for existing clients adding a new community.
Filter / RouterAccelo: Search Company
4
ACTION — Accelo: Create Company (if new)
Map fields: Company Name = community name, Primary Contact = reporting contact name/email, Billing Contact = billing contact name/email (from form). Tag with community type (Multi-Family, Homebuilding, etc.).
Accelo APIAuto
5
ACTION — Accelo: Create Retainer + Service Lines
Use the services selected in the form to look up the correct Accelo Template IDs and create the retainer. Map: start date = form go-live date. Services: Paid Search (396), Paid Social (369), SEO (367), Email (368), ILS (374), Media Planning (373), Display (custom ID). Output: Accelo Retainer ID for use in later steps.
Accelo APIAuto
6
ACTION — Accelo: Create Project with Tasks
Create the onboarding project under the new company. Auto-assign tasks based on services selected using template logic. Map assignees from P11 role lookup table. Set due dates based on start date + standard SLA offsets (e.g., admin setup = Day 2, campaign build = Day 9).
Accelo APIVerify
7
ACTION — Basecamp: Create Project Page
Using the Basecamp API + Webhook, create a new project titled [Community Name] — Onboarding. Auto-generate to-do lists per selected service (Paid Search, Paid Social, SEO, etc.). Add Accelo project link as a pinned message. Post welcome note to Message Board with key dates.
Basecamp APIAuto
8
ACTION — Dropbox: Create Public Client Folder
Create folder at path: /P11 Public/Clients/[Community Name]/ with standard subfolders: Assets, Logos, Brand Guide, Creative Campaigns, Ad Copy, Demographic Profiles, Other. Add access emails from Softr user list. Capture the public shareable link for use in Step 10.
Dropbox APIAuto
9
ACTION — Google Drive: Create Onboarding + Active Folders
Create two folders in P11's shared Google Drive: /Onboarding/[Community Name]/ and /Active/[Community Name]/. Make a copy of the Campaign Structure / Keywords / Ad Copy Google Sheet template inside Onboarding folder. Make a copy of the Media Schedule template inside Active folder. Capture shareable links for both.
Google Drive APIAuto
10
ACTION — Softr: Create Client Portal User + Community
Create a new Softr user for the reporting contact (name + email from form). Create community record with: community name, type, services (from form), go-live date, Dropbox folder link (from Step 8), Accelo project link (from Step 6). This populates the client-facing onboarding dashboard automatically.
Softr APIAuto
11
ACTION — Gmail / Email: Send Client Kickoff Email
Auto-populate the kickoff email template and send to the reporting contact. Include: Softr portal link + login instructions, Ways of Working doc link, Dropbox folder link, process overview (7 stages with timeline), next step (grant platform access with guide links). Route draft to Account Manager for final review before sending, OR send automatically if AM is pre-approved to skip review.
Gmail / EmailAM Reviews Draft
12
ACTION — Slack: Notify P11 Team (Internal)
Post to the P11 internal Slack channel (e.g., #new-clients or #digital-team): "🎉 New client onboarded: [Community Name] ([Type]). Go-live: [Date]. Services: [list]. Acct Mgr: [name]. Basecamp → [link]." This replaces the manual "notifying the team" step from the current process.
Slack APIAuto

Implementation Priority — What to Build First
Priority Automation Tool(s) Effort Impact Notes
1 — Now Form submission → Basecamp project auto-creation Zapier + Basecamp API Medium 🔥 Very High Eliminates the biggest manual bottleneck. Use Zapier's native Basecamp integration.
1 — Now Form submission → Accelo company + retainer creation Zapier + Accelo API Medium 🔥 Very High Accelo has a Zapier integration. Service template IDs are already documented.
1 — Now Dropbox folder auto-creation with subfolders Zapier + Dropbox API Low 🔥 High Zapier has native Dropbox folder creation. Sub-folder creation requires a multi-step Zap or Make.com.
2 — Soon Kickoff email auto-draft to Account Manager Zapier + Gmail Low High Create a Gmail draft (not auto-send) so AM can personalize before sending. Reduces 30 min of work to 5 min.
2 — Soon Softr client portal auto-setup Zapier + Softr API Medium High Softr has a REST API. Create user + community + services record. Check Softr docs for field mapping.
2 — Soon Slack new client notification to P11 team Zapier + Slack Very Low Medium Simplest Zap in the list. Build this first as a "quick win" to prove automation value to the team.
3 — Phase 2 Google Drive folder auto-creation with template copies Make.com + Google Drive Medium-High Medium Requires Make.com (better Google Drive support than Zapier). Needs "copy file" + "move to folder" steps.
3 — Phase 2 Access reminder sequence (48hr delay) Zapier + Gmail Medium Medium Requires Zapier's Delay feature (Zapier Professional plan). High ROI for reducing AM follow-up time.
3 — Phase 2 Stage advancement → client tracker update Make.com / Softr API + Basecamp High High Most complex automation. Requires bi-directional sync between Basecamp task completions and Softr stage field.
Important Setup Notes
🔑
API Keys needed: You'll need active API credentials for Jotform, Accelo, Basecamp, Dropbox, Google Drive, Softr, and Slack. Store all credentials in a secure password manager (1Password or similar) — never hardcode in Zapier step names or descriptions.
⚠️
Jotform vs. Softr form: If you migrate the intake questionnaire from Jotform to a native Softr form, you'll need to update Zap 1 Trigger to watch the Softr form webhook instead. The field mapping will change. Decide on the form tool before building automations.
💡
Zapier vs. Make.com: Zapier is easier to set up but more expensive at scale. Make.com (formerly Integromat) is more powerful for complex multi-step logic, branching, and Google Drive operations. Recommendation: start with Zapier for Phases 1–2, then evaluate Make.com for Phase 3 automations.
✅
Sensitive data: Payment / billing info should NOT pass through Zapier. Collect billing details separately via a secure form (Stripe, Typeform with encryption, or direct to Admin). Instruct clients never to email credit card or banking information.
🔮
Phase 3 moonshot: Once the core automation is running, consider building a Notion database as the "brain" that all tools feed into — community name, Accelo ID, Basecamp ID, Softr ID, Google Drive links, stage, services, go-live date — all in one place. This becomes the master record that powers dashboards, reporting, and client communication at scale.
